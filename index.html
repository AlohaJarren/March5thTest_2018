What are Wireframes?

Wireframes are 2D drawn illustrations of how a specific webpage on a website is supposed to look like. It shows the exact placements of a page's features and outlines placements of what is to be expected in the webpage creation process. We use wireframes as a visual guide for the overall structure of what the webpage of a website is supposed to look like. The point of wireframes is to help us on primarily website structuring and placing certain elements on where it should be on the web page.

Git Workflow

When working with a group on a certain project placed in Github, one individual creates a repository and shares it with their group. From there, every individual creates their own branches through command ~git checkout -b [branch_name]. Every time someone commits a change, an individual may create a pull request to be approved by the creator of the repository. If the pull request is accepted, the master branch inherits the changes and every individual must update their branch's code to be worked on to avoid potential errors in GitHub.

Ways to Create Responsive Web Design

  • Creating viewports: Establishing meta viewport tags for browser to interpret and adjust computer width and scale adjustment of a web page.
  • Using %s: By applying percentages instead of pixels in CSS code, images would change proportionally to the browser when individual shrinks or enlarges the page.
  • Mobile First Development: Working on a website starting from mobile phones to be more easily adapted onto computers later on.
  • Flexbox: The process of creating flex containers through HTML and CSS for purposes of adjusting text width and height to make the best use of webpage spacing

Arrays and Loops

Arrays are objects that are capable of storing two or more values into one single variable, in which values could easily be added or removed. To add, loops are used in JavaScript to run a block of code for a set amount of times until the code's conditions are met.
